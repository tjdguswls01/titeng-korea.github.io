02 / GitHub VS Code + Copilot

AI 시대에 GitHub 사용하기

Repository, Issue, Pull Request, Release의 기본 흐름을 먼저 잡고, VS Code와 GitHub Copilot을 연결해 실제 개발 보조 흐름으로 확장합니다.

Product Copilot 참고 영상

Session flow

  1. 1GitHub 저장소 구조와 파일 변경 추적
  2. 2Issue와 Pull Request로 작업 단위 관리
  3. 3VS Code에서 commit, push, review 흐름 확인
  4. 4GitHub Copilot을 코드 작성 보조로 연결

Repository

Product, 문서, 샘플 코드 저장소를 구분하고 변경 이력을 관리합니다.

Issue & PR

요구사항, 버그, 리뷰, 승인 흐름을 작업 단위로 정리합니다.

Release

SDK, 문서, 발표 자료의 버전과 배포 지점을 명확하게 남깁니다.

발표 포인트

  • GitHub는 단순 저장소가 아니라 협업 기록과 의사결정 흐름을 남기는 도구로 설명합니다.
  • Copilot은 코드를 대신 맡기는 도구가 아니라 초안 작성과 반복 작업을 줄이는 보조 도구로 위치시킵니다.
  • VS Code에서 변경 파일을 확인하고 리뷰 가능한 단위로 나누는 흐름을 보여줍니다.
