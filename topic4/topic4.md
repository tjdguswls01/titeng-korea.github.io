너는 웹 프론트엔드 개발자이자 전문 기술 프레젠테이션 디자이너야. 내가 제공하는 고밀도 기술 기획안과 대본을 바탕으로, 웹브라우저에서 바로 열어서 '50분간 단독 발표'할 수 있는 마스터피스 수준의 단일 HTML 슬라이드 파일(`session2.html`)을 생성해줘.

외부 라이브러리(예: Reveal.js)를 CDN으로 연동하거나, 순수 HTML/CSS/JavaScript만을 사용하여 실제 파워포인트처럼 방향키(좌우)나 스페이스바로 슬라이드가 넘어가고 가독성이 극대화된 모던한 디자인으로 구현해줘.

  * 메인 배경: 순수 블랙 또는 극도로 어두운 차콜 계열 (#05070b 또는 #0a0f1d)

- 대본: 개발자의 집중력을 깨뜨리는 주범은 창 전환입니다. 이 플러그인은 모든 디자인 워크플로우를 에디터 내부 단일 파이프라인으로 묶어 집중력의 끈을 유지해 줍니다.

[슬라이드 13] 손과 키보드의 일체화 (Keyboard-Driven Development)
- 내용: 손이 마우스로 이동할 때 발생하는 초 단위의 지연과 뇌의 흐름 끊김을 차단하고 키보드 순정 속도 극대화

- 대본: 키보드 타건의 속도가 곧 생각의 속도가 되는 세계입니다. 명령어 한 줄로 수십 개의 파일을 제어하는 셸 환경에 AI가 결합되었을 때의 무시무시한 화력을 보시죠.

[슬라이드 14] GitHub Copilot CLI의 메커니즘과 인터페이스

- 대본: 리눅스 서버를 관리하다 보면 복잡한 awk, sed 문법이나 docker 네트워크 관련 옵션이 기억나지 않아 헤매곤 합니다. 이제 터미널에 대고 우리말로 물어보면 됩니다.

[슬라이드 15] 실전 예시: 복잡한 인프라 명령어 조합 자동 생성
- 내용: "8080 포트를 점유 중인 프로세스를 찾아서 강제 종료하고 로그 파일의 마지막 100줄을 아카이빙해줘" 명령의 CLI 변환 과정과 실제 셸 출력문
- 대본: 방금 말한 긴 요구사항을 터미널 비서가 정확한 파이프라인(`|`) 기호와 옵션 조합의 셸 스크립트로 즉시 작성해 주며, 엔터 한 번으로 실행까지 마칩니다.

[슬라이드 16] Part 4. 자율 주행형 CLI 에이전트의 충격
- 내용: 단순 답변기 역할을 넘어, 개발자의 터미널 권한을 위임받아 스스로 판단하고 코드를 고치는 에이전트 시스템
- 대본: 네 번째 파트이자 이번 발표의 하이라이트입니다. 단순히 코드를 추천해 주는 비서를 넘어, 지시만 내리면 스스로 생각하고 컴퓨터를 제어하는 자율형 에이전트들을 파헤쳐 보겠습니다.

[슬라이드 17] Claude Code의 아키텍처와 도구 사용(Tool Use)
- 내용: Anthropic의 Claude Code가 터미널 환경에서 파일 읽기/쓰기, grep 검색, bash 커맨드 실행 도구를 자율적으로 호출하는 루프 구조
- 대본: 최근 등장한 클로드 코드는 충격적입니다. 단순 답변을 넘어 에이전트 권한을 가집니다. 터미널에 "버그 수정하고 빌드해서 에러 없는지 검증해줘"라고 하면, 스스로 관련 파일을 열어 고치고 컴파일을 돌려 성공 여부까지 보고합니다.

[슬라이드 18] 에이전트 기반 자율 코딩의 작동 메커니즘
- 내용: 목표 부여 -> 환경 스캔 -> 패치 코드 작성 -> 컴파일 테스트 -> 에러 피드백 -> 자율 수정 완료로 이어지는 에이전트의 내부 사고방식(ReAct 프레임워크)
- 대본: 에이전트는 하나의 명령을 받으면 그것을 세부 태스크로 쪼갭니다. 그리고 각 태스크를 실행하면서 실패하면 왜 실패했는지 스스로 이유를 찾아 다음 단계에 반영하는 고도의 자율 루프를 돕니다.

[슬라이드 19] Codex CLI를 활용한 대규모 리팩토링 실전
- 내용: "레거시 자바스크립트 프로젝트 전체를 타입스크립트로 마이그레이션해줘" 지시 시 수십 개 파일의 타입을 추론하여 자동 변환하는 실제 코드 디프(Diff) 연출
- 대본: "이 프로젝트 전체를 타입스크립트로 바꾸고 타입 에러 다 잡아줘" 같은 거대한 리팩토링도 명령 한 줄이면 에이전트가 밤새 지치지 않고 수십 개의 파일을 돌아다니며 완벽하게 컴파일 가능한 상태로 고쳐놓습니다.

[슬라이드 20] 킬러 사용법 1: 유기적 멀티 파일 동시 리팩토링
- 내용: 데이터베이스 스키마 변경 시, 이와 연동된 ORM 모델, DTO, 서비스 로직, API 엔드포인트 파일 5개를 에이전트가 종속성을 추적하여 동시 수정하는 흐름도
- 대본: "유저 테이블에 이메일 필드 추가하고 연관된 API 전부 수정해줘"라고 터미널에 치면, Claude Code가 프로젝트 전체를 스캔하여 5개 파일을 알아서 한 번에 깔끔하게 고쳐놓습니다.

[슬라이드 21] 킬러 사용법 2: 자율형 빌드 컴파일 및 런타임 에러 셀프 패치
- 내용: 에이전트가 스스로 빌드 스크립트를 수행하고, 터진 스택 트레이스(Stack Trace) 에러 로그를 분석하여 임포트 누락 및 문법 오류를 자율 수정하는 빌드 자동화 데모
- 대본: 빌드를 돌리다 컴파일 에러가 나면, 에이전트가 에러 로그를 읽고 '아, 이 파일의 임포트가 빠졌구나'를 스스로 깨달은 뒤 코드를 고치고 다시 빌드를 돌려 성공 판정까지 내립니다.

[슬라이드 22] Git 워크플로우 자동화와 시니어 수준의 Commit 작성
- 내용: 코딩 종료 후 `codex git commit` 커맨드 입력 -> 소스코드 변경 내역(Diff)을 AI가 초고속 분석하여 세련된 커밋 메시지 작성 및 원격 푸시 자동화
- 대본: 코딩이 끝나면 터미널에 `codex git commit` 한 줄만 치면 됩니다. 변경 사항을 문맥적으로 요약하여 현업 컨벤션에 맞는 완벽한 깃 커밋 메시지를 자동으로 작성하고 리포지토리에 푸시합니다.

[슬라이드 23] 터미널 내 스트리밍 예외 로그 실시간 추적 패치
- 내용: 로컬 서버 구동 중 콘솔에 쏟아지는 런타임 NullPointer 예외 등을 CLI 에이전트가 실시간 인터셉트하여 방어 로직 패치를 즉각 제안하는 과정
- 대본: 런타임에 에러 덤프가 쏟아지면, 터미널에 상주하는 에이전트가 스택 트레이스를 실시간 캡처하여 원인을 진단하고 그 자리에서 코드를 패치합니다.

[슬라이드 24] 휴먼 개발자의 새로운 정체성: 코드 감사관 (Code Auditor)
- 내용: 단순 타이핑 노동자(Coder)에서 에이전트가 자율 제출한 소스코드 디프(Diff) 내역을 최종 검토, 보안성 검증, 승인(Approve)하는 아키텍트 거버넌스의 정립
- 대본: 이제 우리는 타이핑하는 노동자가 아닙니다. 에이전트가 자율적으로 고쳐놓은 코드의 변경 사항을 꼼꼼히 리뷰하고 최종 승인 도장을 찍는 감사관의 역할을 수행하게 됩니다.

[슬라이드 25] 실전 트러블슈팅: 자율 에이전트의 폭주(Infinite Fix Loop) 제어
- 내용: 에이전트가 코드를 고치다 또 다른 버그를 내고, 그걸 고치려다 컴파일러를 터트리는 무한 수정 루프 현상 분석 및 `Ctrl+C` 강제 개입 타이밍 가이드
- 대본: 에이전트가 간혹 코드를 고치다 꼬여서 무한 수정 루프에 빠지는 폭주 현상이 일어날 수 있습니다. 이를 방지하기 위한 최대 실행 스텝 설정과 수동 제어 개입 타이밍을 숙지해야 합니다.

[슬라이드 26] 엔터프라이즈 보안 가이드: 에이전트 권한 격리와 격리 범위 설정
- 내용: 사내 환경변수(`.env`), 프라이빗 키, DB 크레덴셜 폴더를 에이전트의 탐색 및 읽기 권한에서 제외하기 위한 `.agentignore` 설정 파일 구축 예시
- 대본: 에이전트에게 너무 강력한 자율 권한을 주면 중요 설정 파일이나 DB 비밀번호가 유출되거나 오염될 수 있습니다. 반드시 `.agentignore` 같은 설정을 통해 접근 권한 영역을 통제해야 합니다.

[슬라이드 27] 도구별 생산성 비교 매트릭스 (Total Matrix)

[슬라이드 28] 10배 개발자(10x Developer)의 일일 워크플로우 시나리오
