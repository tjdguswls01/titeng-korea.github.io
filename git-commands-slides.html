Git Command Quick Guide

실무에서 자주 쓰는
Git 명령어 10가지

상태 확인부터 브랜치, 커밋, 원격 저장소 동기화, 되돌리기까지 한 흐름으로 정리합니다.

전체 흐름

Git 명령어는 외우는 순서보다
작업 흐름으로 묶으면 쉽습니다.

1status현재 변경 파일과 stage 상태를 먼저 확인합니다.
2diff무엇이 바뀌었는지 줄 단위로 검토합니다.
3add이번 커밋에 넣을 변경만 고릅니다.
4commit변경 의도를 메시지와 함께 기록합니다.
5push / pull원격 저장소와 변경을 주고받습니다.
1. 상태 확인

git status는 작업 전후에 가장 먼저 보는 명령어입니다.

수정된 파일, 새 파일, 삭제된 파일, stage 여부를 확인합니다. AI나 자동화 도구가 만든 변경을 검토할 때도 첫 단계입니다.

git status

# 짧게 보고 싶을 때
git status -sb

## main...origin/main
 M src/app.js
?? .env.local
2. 변경 내용 보기

git diff는 커밋 전 검토의 핵심입니다.

추가된 줄보다 삭제된 줄을 더 주의해서 봅니다. 의도하지 않은 설정 변경, 보안 정보, 테스트 삭제가 없는지 확인합니다.

git diff

# staged 변경만 보기
git diff --staged

# 특정 파일만 보기
git diff src/app.js
3-4. 커밋 만들기

git add와 git commit은 한 세트입니다.

`add`는 이번 커밋에 넣을 변경을 고르는 과정이고, `commit`은 그 변경을 설명 가능한 기록으로 남기는 과정입니다.

git add src/app.js
git commit -m "fix: 로그인 실패 메시지 표시"

# 모든 변경을 한 번에 stage
git add .

# 일부 변경만 대화식으로 stage
git add -p
5-6. 브랜치 작업

실험과 기능 작업은
브랜치에서 분리합니다.

`git branch`로 목록을 보고, `git switch`로 이동합니다. 새 작업은 main에서 바로 하지 말고 새 브랜치를 만든 뒤 시작하는 편이 안전합니다.

main
feature/login
브랜치 명령어

브랜치는 현재 작업의
경계를 명확하게 만듭니다.

작업 목적이 이름에 드러나게 만들면, 나중에 PR과 커밋 히스토리를 읽기 쉬워집니다.

git branch

# 새 브랜치 생성 후 이동
git switch -c feature/login-message

# 기존 브랜치로 이동
git switch main
7-8. 원격 저장소 동기화

git pull과 git push는 원격 저장소와 주고받는 명령입니다.

작업 시작 전에는 최신 변경을 받고, 커밋 후에는 내 변경을 올립니다. 충돌이 나면 자동 해결보다 diff 확인이 먼저입니다.

git pull

# 현재 브랜치를 원격에 처음 올릴 때
git push -u origin feature/login-message

# 이후에는 보통 이것만
git push
9. 기록 확인

git log는 프로젝트의 변경 이력을 읽는 명령입니다.

누가, 언제, 어떤 의도로 변경했는지 확인합니다. 문제를 추적하거나 되돌릴 커밋을 찾을 때 자주 씁니다.

git log

# 실무에서 보기 좋은 형태
git log --oneline --graph --decorate --all

* 3f4a9c2 fix: login message
* 91e20b1 feat: add user page
10. 안전하게 되돌리기

git restore는 아직 커밋하지 않은 변경을 되돌릴 때 씁니다.

파일 단위로 작업 내용을 버릴 수 있으므로 신중하게 사용합니다. 실행 전 `git diff`로 버릴 내용을 반드시 확인합니다.

# 파일 변경 되돌리기
git restore src/app.js

# stage에서만 내리기
git restore --staged src/app.js

# 주의: 되돌린 변경은 쉽게 복구하기 어렵습니다.
10개 명령어 요약

최소한 이 10개만 익혀도
대부분의 기본 작업을 처리할 수 있습니다.

git status현재 변경 상태 확인
git diff커밋 전 변경 내용 검토
git add커밋할 변경 선택
git commit변경을 기록으로 저장
git branch브랜치 목록과 현재 위치 확인
git switch브랜치 이동 또는 생성
git pull원격 변경 가져오기
git push내 커밋을 원격에 올리기
git log커밋 히스토리 확인
git restore커밋 전 변경 되돌리기
추천 루틴

작업할 때는 이 순서로
명령어를 연결해 쓰면 됩니다.

상태 확인, 변경 검토, 필요한 것만 stage, 메시지 작성, 원격 반영. 이 루틴이 Git 실수 대부분을 줄입니다.

git status -sb
git switch -c feature/my-work
git diff
git add -p
git commit -m "feat: add useful change"
git pull
git push -u origin feature/my-work
